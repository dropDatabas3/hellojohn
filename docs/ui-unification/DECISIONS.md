# UI Unification Decisions

> If we break a rule, we log it here.

## Closed Decisions

### 1. Separate Palettes (Option B)
- **Decision:** Use distinct palettes for Dark and Light modes instead of simple mapping.
- **Why:** To achieve maximum visual quality ("High-Fidelity") in both modes.

### 2. Execution Order
- **Decision:** Dark Mode first, then Light Mode.
- **Why:** Dark mode defines the "truth" of the design system (depth, clay effects).

### 3. Animation Speed
- **Decision:** Midpoint (Professional + Visible).
- **Why:** Needs to feel responsive but premium.

### 4. UI Kit Strategy
- **Decision:** Full replacement of UI kit; Radix allowed only as headless base.
- **Why:** To ensure strict adherence to the new Design System tokens and aesthetics.

### 5. Visual Changes
- **Decision:** Visual changes are allowed and expected.
- **Why:** Consistency of the new system takes priority over backward compatibility.

### 6. Workflow
- **Decision:** Single-dev, low ceremony.
- **Why:** Speed and efficiency. Docs and checkpoint commits replace PRs/branches.
